# FixMzansi MVP Scope

## Goal
Build the smallest product that proves a homeowner can find, trust, and book a solar professional faster and more reliably than WhatsApp referrals.

## Core user problem
Cape Town customers need solar help, but they struggle to:
- find reputable installers
- compare pricing transparently
- verify quality and credentials
- avoid low-quality or unresponsive providers
- get fast booking and reliable communication

## MVP promise
"Trusted solar installers in Cape Town, instantly discoverable and bookable."

## MVP target market
- City: Cape Town
- Category: solar installation and solar-related home energy services
- Target users: homeowners, landlords, property managers, small businesses
- Supply side: solar installers, electricians who do solar, inverter and battery specialists

## MVP must include
1. Consumer request flow
   - service type
   - job description
   - photos
   - suburb / location
   - urgency

2. Pro directory
   - verified profile
   - service area
   - rating
   - completed jobs
   - contact / booking button

3. Matching
   - show 3 to 5 relevant pros
   - rank by proximity, availability, rating, verification status

4. Booking
   - request a time slot
   - confirm via web
   - notify pro and consumer

5. Trust layer
   - manual vetting at first
   - certificate upload
   - verified badge
   - completed-job reviews only

6. Basic payments
   - start with reservation or deposit flow if full escrow is too heavy
   - capture the intent to pay digitally from day one

7. Admin tools
   - add or approve pros
   - review jobs
   - resolve disputes
   - edit listings

## Explicitly out of scope for MVP
- BNPL
- mobile app
- insurance or warranties
- complex AI pricing engine
- multi-city launch
- advanced automation

## MVP success criteria
The MVP is successful if:
- users can find a pro in under 2 minutes
- pros can receive bookings without friction
- some jobs complete successfully
- users trust the platform enough to come back
- the product can expand later without rebuilding everything

## MVP principle
If a feature does not improve trust, booking speed, or conversion, it does not belong in the MVP.
