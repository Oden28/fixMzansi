# FixMzansi: First 10 Features to Build

## Goal
Build the smallest high-leverage set of features that creates trust, booking flow, and repeat usage.

## 1. Home / landing page
Purpose:
- explain the product quickly
- capture demand
- guide users into the request flow

## 2. Service request form
Purpose:
- collect job type, location, photos, urgency, and description
- turn intent into a structured lead

## 3. Pro profile page
Purpose:
- display trust signals
- show work history
- show service area
- give consumers confidence

## 4. Pro verification badge system
Purpose:
- mark pros who are verified
- create a trust layer that differentiates the platform

## 5. Matching / recommendations list
Purpose:
- show the most relevant pros for a request
- rank by proximity, availability, rating, and verification

## 6. Booking flow
Purpose:
- let the consumer choose a pro
- select a time
- confirm the job

## 7. Admin dashboard
Purpose:
- manage listings
- approve pros
- edit requests
- monitor jobs
- handle exceptions manually

## 8. Review system
Purpose:
- collect only post-job reviews
- build trust over time
- make quality visible

## 9. Notifications / WhatsApp updates
Purpose:
- keep both sides informed
- reduce no-shows
- increase responsiveness

## 10. Basic payments flow
Purpose:
- prove willingness to pay digitally
- reduce cash friction
- lay the foundation for escrow and future finance products

## Build order
1. landing page
2. request form
3. pro profiles
4. verification badge
5. matching list
6. booking flow
7. admin dashboard
8. review system
9. notifications
10. payments
