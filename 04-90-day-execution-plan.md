# FixMzansi 90-Day Execution Plan

## Goal
Launch a credible, focused version of FixMzansi and prove real demand before expanding.

## Days 1-10: Strategy and setup
- finalize the wedge category
- choose the first city / suburb cluster
- define the target customer profile
- create the brand and messaging
- secure domain and project workspace
- write the product requirements
- build the basic operating dashboard

Deliverables:
- mission and positioning
- MVP definition
- initial wireframes
- launch checklist
- supply onboarding script

## Days 11-20: Supply acquisition
- manually recruit the first 20-50 pros
- verify credentials
- collect photos, bios, service areas, pricing expectations
- identify a few exceptional pros to feature prominently

Deliverables:
- verified pro list
- onboarding pipeline
- vetting checklist
- initial service categories

## Days 21-35: Core product build
- build landing page
- build request form
- build pro profile pages
- build matching logic
- build booking flow
- build admin review tools

Deliverables:
- working MVP on web
- manual admin backend
- usable consumer journey

## Days 36-45: Trust and communications
- add review capture
- add WhatsApp updates or manual messaging workflows
- add notifications
- refine dispute handling
- polish trust signals

Deliverables:
- trust layer
- communication layer
- review system

## Days 46-60: Beta launch
- invite first users
- run real jobs
- track conversion and completion
- interview consumers and pros
- fix friction points fast

Deliverables:
- first completed jobs
- real customer feedback
- bug list
- product iteration backlog

## Days 61-75: Growth experiments
- run local SEO pages
- test paid acquisition
- test referral loops
- test suburb-focused landing pages
- test WhatsApp sharing loops

Deliverables:
- acquisition data
- conversion data
- early growth signals

## Days 76-90: Tighten the business
- measure repeat usage
- measure supply retention
- improve pricing structure
- decide where to expand next
- prepare investor / partner narrative

Deliverables:
- traction dashboard
- updated roadmap
- next-market decision
- funding story

## 90-day success criteria
- enough supply to answer real demand
- first repeat bookings
- strong user trust signals
- evidence of a repeatable customer acquisition path
- clear next step toward scale

## 90-day rule
Do not expand too early. The first 90 days are about proof, not breadth.
