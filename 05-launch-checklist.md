# FixMzansi Launch Checklist

## Goal
Make launch operationally ready, trustworthy, and easy to iterate.

## Brand and positioning
- [ ] domain secured
- [ ] brand name locked
- [ ] value proposition written clearly
- [ ] launch messaging approved
- [ ] basic visual identity defined

## Market focus
- [ ] first city selected
- [ ] first service category selected
- [ ] target customer defined
- [ ] target pro profile defined
- [ ] pricing assumptions documented

## Supply side readiness
- [ ] first pros recruited
- [ ] verification process defined
- [ ] trade certificate collection method ready
- [ ] profile creation workflow ready
- [ ] response expectations documented

## Product readiness
- [ ] landing page live
- [ ] request form works
- [ ] pro profiles display correctly
- [ ] booking flow tested
- [ ] admin dashboard available
- [ ] review capture available

## Trust and safety
- [ ] verification badge system defined
- [ ] dispute resolution process defined
- [ ] terms and conditions drafted
- [ ] privacy policy drafted
- [ ] support contact channel live

## Operations
- [ ] manual fallback process defined
- [ ] pro communication process ready
- [ ] consumer communication process ready
- [ ] job status tracking defined
- [ ] escalation path ready

## Metrics
- [ ] bookings tracked
- [ ] conversion tracked
- [ ] response time tracked
- [ ] completion rate tracked
- [ ] repeat usage tracked
- [ ] review rate tracked

## Go-to-market
- [ ] WhatsApp sharing flow ready
- [ ] SEO landing pages ready
- [ ] first paid tests prepared
- [ ] referral system ready
- [ ] launch announcement drafted

## Post-launch discipline
- [ ] user feedback loop active
- [ ] weekly review cadence set
- [ ] bug list maintained
- [ ] roadmap prioritized by revenue and trust impact
- [ ] expansion criteria defined

## Launch rule
If the product cannot reliably deliver trust and bookings in one small geography, it is not ready to scale.
